# Contributing to Swytchcode Examples

First off, thank you for considering contributing to the Swytchcode Examples repository! We welcome contributions from developers of all backgrounds.

## How to Contribute

1. **Fork the Repository**: Start by forking this repository to your own GitHub account.
2. **Clone Locally**: Clone your fork locally to your development environment.
3. **Create a Branch**: Create a new branch for your feature or bugfix (e.g. `feat/new-langgraph-demo`).
4. **Make Your Changes**: Add your new example project or improve an existing one. Please ensure it aligns with the existing structures.
5. **Test Your Changes**: Verify that your demo runs correctly locally and includes a well-written `.env.example` and `README.md`.
6. **Submit a Pull Request**: Submit a PR against the `main` branch of this repository. Provide a clear description of what your example does.

## Guidelines for New Examples
- Ensure all API keys and secrets are removed.
- Include an architecture overview or diagram in your `README.md`.
- Keep dependencies as lean as possible.
- Adhere to the core principles of idempotency and auditability that Swytchcode guarantees.

We will review your PR as quickly as possible!
